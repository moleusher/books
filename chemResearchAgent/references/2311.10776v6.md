# Chemist-X: Large Language Model-Powered Agent for Recommending Reaction Conditions in Chemical Synthesis and Autonomous Laboratories

**URL:** https://arxiv.org/html/2311.10776v6

## Authors

Kexin Chen, Jiamin Lu, Junyou Li, Xiaoran Yang, Yuyang Du, Kunyi Wang, Qiannuan Shi, Jiahui Yu, Lanqing Li, Jiezhong Qiu, Jianzhang Pan, Yi Huang, Qun Fang, Pheng Ann Heng, Guangyong Chen

## Abstract

This study presents Chemist-X, an AI agent that automates reaction condition optimization (RCO) in chemical synthesis using retrieval-augmented generation (RAG) and AI-controlled wet-lab experiments. The system emulates how human chemists approach RCO by employing a novel RAG scheme to query molecular and literature databases, narrowing the search space. A computer-aided design (CAD) tool, developed through an LLM-supervised programming interface, is then leveraged. With updated chemical knowledge from RAG and CAD tool capabilities, the agent outperforms conventional RCO AIs restricted to fixed training data. Finally, Chemist-X interacts with physical lab equipment through an automated robotic system for validation without human intervention. The authors report that "results of our automatic wet-lab experiments, achieved by fully LLM-supervised end-to-end operation with no human in the lope, prove Chemist-X's ability in self-driving laboratories."

## 1 Introduction

The paper situates itself within the broader "AI for chemistry" movement aiming to free human labor through AI-supervised chemical reaction platforms. These systems employ machine learning and data-driven methods to orchestrate synthetic routes and identify optimal conditions. The authors note that "chemical robots equipped with integrated AIs will autonomously design and refine chemical reactions" in the foreseeable future.

Traditional AI systems fall short because they are confined to knowledge from training data and show limited generalization for unknown reactions. Recent advances in RAG-AI address this by combining retrieval-based methods with generative AIs like LLMs. The paper focuses on RAG for reaction condition optimization (RCO), which the authors describe as "equally important as retrosynthesis but receives less attention in synthetic research."

The three-phase pipeline works as follows:

- **Phase One:** The LLM generates Python code to search chemical literature and molecule databases for reaction information about the target molecule and its analogs, narrowing the chemical space.
- **Phase Two:** The LLM-powered agent uses CAD tools to select promising conditions from the refined space.
- **Phase Three:** The agent controls auto-lab equipment via LLM-generated scripts to conduct wet-lab experiments validating the recommended conditions.

This "retrieve-recommend-validate" pattern mirrors how a human expert would address RCO for an unfamiliar molecule.

**Phase One details:** The system applies a hierarchical matching strategy using exact SMILES matching against a private database built on Amazon EC2 in collaboration with CAS. If the compound is not found, PubChem API generates similar molecules for iterative searching. The authors introduce "top match slice" (TMS)—the portion of API documentation with the highest semantic similarity to the problem description—to enhance in-context learning for code generation.

**Phase Two details:** The agent selects from CAD algorithms via API programming. The paper develops a novel ML model using "chemistry-informed molecular graph (CIMG)" as a molecular descriptor and "supervised contrastive learning (SCL)" for fingerprint generation. This "coarse-label SCL (CL-SCL) fingerprint" replaces numerical yield descriptions with categorical labels to handle noise.

**Phase Three details:** A general computer control framework simulates human manipulation of software interfaces using LLM reasoning, pyautogui, key frame detection, mouse-click detection, and OCR. The system is implemented on the iChemFoundry (IC) platform, which "successfully automates reaction pretreatment, including solid dosing and synthesis under inert gas protection."

## 2 Results and Discussions

The paper demonstrates Chemist-X on a Suzuki reaction targeting 6-(1-methyl-1H-indazol-4-yl) quinoline (SMILES: CN1C2=CC=CC(C3=CC=C4N=CC=CC4=C3)=C2C=N1).

### 2.1 Phase One Unit Tests

The agent splits PubChem's API documentation into eight slices and uses OpenAI's text-embedding-ada-002 to convert document slices and the query prompt into semantic vectors. Two similarity metrics were tested: Cosine Similarity and L2 Similarity. Across both metrics, "slice #3 achieves the highest semantic similarity regardless of the metric applied." This slice pertained to PubChem's "similar molecules inquiry function" and included a practical programming example.

The TMS-ICL approach was compared against three alternatives: zero-shot learning, ICL with complete documentation, and ICL with a random slice. Ten trials using GPT-4 showed that "our TMS-ICL method consumes slightly more computational resources than the random slice approach yet delivers a remarkably higher success rate than all three alternatives."

### 2.2 Phase Two Unit Tests

The CL-SCL fingerprint was trained on a Buchwald-Hartwig reaction dataset and tested on a Suzuki-Miyaura dataset. The authors used maximum observed yield (μ_N) with batch size N as the evaluation metric.

**Benchmarking:** CL-SCL was compared against DRFP and Mordred fingerprints across random forest (RF), tabular transformer (TT), and Xgboost (XGB) models at various batch sizes. The CL-SCL fingerprint "consistently delivers superior reaction condition recommendations" regardless of batch size or ML model.

**Ablation study** (batch size = 5) tested two variants: removing the CL-SCL network and replacing CIMG with a simple concatenation encoder. The results showed that "both the CIMG encoder and the CL-SCL network are important components" — the original method had significantly higher μ_5 across all ML models.

### 2.3 Phase Three Wet-Lab Validations

#### 2.3.1 Experimental Setup

The initial chemical subspace for the Suzuki reaction included four dimensions (reactant, ligand, base, solvent) with 10,000 possible reactions. After Phase One, the agent identified K2CO3 as optimal base and dioxane/H2O as most suitable solvent from existing literature, reducing the space to 100 configurations across two dimensions (reactants and ligands). Phase Two then made final recommendations. The process was repeated three times with different random seeds, plus random selections as a benchmark.

#### 2.3.2 Experimental Details and Discussions

All reactions were performed in the Unchained Lab's optimization sampling reactor (OSR) with heating, pressurization, continuous stirring, and inert gas protection. Product purification used column chromatography on silica gel 200–300 mesh with HPLC-MS identification. Yield analysis employed 4-chloro-6,7-dimethoxyquinoline as internal standard via HPLC.

A typical procedure: 6-bromoquinoline (1b, 312 mg), 1-methyl-4-(4,4,5,5-tetramethyl-1,3,2-dioxaborolan-2-yl)-1H-indazole (2d, 387 mg), Pd(OAc)2 (16.9 mg), potassium carbonate (519 mg), and Amphos (39.8 mg) in dioxane/H2O (9:1, 15 mL) were added to a 20 mL vial. Five reactions ran in parallel per batch, heated at 85°C for 16 hours under nitrogen. Post-treatment involved adding reacted mixture to internal standard solution in methanol for HPLC analysis.

**Results:** All three experimental batches recommended by Chemist-X "reached a μ5 exceeding 90%," while random sampling "only resulted in a μ5 around 52%."

## 3 Method

### 3.1 Phase One: Information Retrieval Using Molecule and Literature Databases

The phase leverages ICL technology, where an LLM forms "short-term memory" based on specific examples rather than requiring parameter optimization. The challenge is automating example selection so the system adapts to different databases without human intervention.

The TMS method works by: dividing the tutorial document into similarly-sized slices, tokenizing each slice, and converting to vector representations using OpenAI's ADA-002 model. Cosine similarity between the question embedding B and each slice embedding A_i is computed as:

di = 1.0 − (Σ A_i(k) · B(k)) / (√(Σ A_i²(k)) · √(Σ B²(k)))

Alternatively, L2 similarity is:

d′_i = 1 / (1 + √(Σ (A_i(k) − B(k))²))

The slice with highest similarity to the question is designated the TMS and used as the ICL example input.

### 3.2 Phase Two: Reaction Condition Optimization with Pre-packaged Fingerprint Tool

Phase One results may be incomplete (missing details like reagents, catalysts, solvents, yield). Phase Two fills this gap with a comprehensive set of potential solutions from the chemical subspace, then identifies the best conditions.

#### 3.2.1 Reaction Data Processing

All chemicals (reactants, ligands, base, solvent, etc.) are encoded using the CIMG descriptor, which incorporates NMR chemical shifts and bond dissociation energy (BDE) as vertex and edge features of a GNN model. For a Suzuki reaction, the concatenated encoding is:

xi = x_Electrophile ⊕ x_Nucleophile ⊕ x_Pd catalyst ⊕ x_Ligand ⊕ x_Base ⊕ x_Solvent

#### 3.2.2 The CL-SCL Network

Reaction yields contain random noise from measurement errors, material purity, and environmental factors. To address this, the authors categorize yields into three types: high-yield, medium-yield, and low-yield ("coarse reaction yield labels"). The SCL loss function is:

L_SCL = Σ_{i=1}^{N} (−1/N_{yi}) Σ_{j≠i, ξ(yj)=ξ(yi)} log( exp(zi·zj/τ) / Σ_{k≠i} exp(zi·zk/τ) )

The post-SCL embedding vector serves as a reaction fingerprint for downstream ML models (random forest, tabular transformer, or Xgboost), packaged within Phase Two's API.

### 3.3 Phase Three: Automated Laboratory Execution with LLM-Supervised Interface Control

Many lab automation platforms lack accessible APIs and rely on GUIs requiring manual interaction. Chemist-X uses an LLM-supervised interface control framework that replicates human interactions with laboratory software.

The system is provided with a demonstration video of a human operator performing three stages:
1. **Initialization:** Launching software and loading modules
2. **Reaction Condition Configuration:** Selecting reagents, parameters
3. **Execution:** Verifying setup and initiating the reaction

Stages 1 and 3 follow fixed operations and are automated by tracking mouse movements and clicks. Stage 2 requires adaptive decision-making based on experiment specifics. A multi-step prompt framework enables the LLM to understand and replicate interactions.

## 4 Conclusion

The paper claims three significant contributions:
1. Through the "search-analyze-recommend" pattern and the novel CL-SCL reaction fingerprint, Chemist-X demonstrates proficiency in automating RCO.
2. Using updated online databases "allows our agent to transcend the limitations of traditional AI systems" confined to training data.
3. The successful application to a Suzuki reaction validated through wet-lab experiments brings "the vision of fully automated chemical discovery one step closer."
